## Design Principles

All components follow these core principles:

- **Type-safe**: Built with TypeScript and strict type checking
- **Accessible**: Following WAI-ARIA standards via Radix UI primitives
- **Composable**: Small, focused components that work together
- **Customizable**: Easily styled using Tailwind CSS utility classes
- **Responsive**: Mobile-first design with dark mode support

## Component Location

Components are organized in the following directory structure:

```bash
src/components/
├── ui/                    # Core UI components (shadcn/ui based)
├── shared/                # Composite and specialized components
├── animated-icons/        # Motion-animated icon components
├── header/                # Header-specific components
└── footer/                # Footer-specific components
```

## Using Components

All components use a consistent API pattern with TypeScript props:

```tsx
import { Button } from "@/components/ui/button"
import { Panel, PanelContent, PanelHeader, PanelTitle } from "@/components/ui/panel"

export default function Example() {
  return (
    <Panel>
      <PanelHeader>
        <PanelTitle>Example Panel</PanelTitle>
      </PanelHeader>
      <PanelContent>
        <p>Your content goes here.</p>
        <Button variant="default" size="default">
          Click me
        </Button>
      </PanelContent>
    </Panel>
  )
}
```

## Styling Utilities

All components use the `cn()` utility function for combining Tailwind classes:

```tsx
import { cn } from "@/lib/utils"

function MyComponent({ className, ...props }) {
  return (
    <div
      className={cn(
        "base-classes here",
        "conditional-classes",
        className
      )}
      {...props}
    />
  )
}
```

This pattern ensures:
- Base styles are always applied
- Custom classes can override defaults
- Type-safe className prop handling
