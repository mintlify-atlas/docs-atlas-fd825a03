Blog posts are written in MDX format and stored in the `src/content/blog/` directory. Each post is a standalone `.mdx` file with frontmatter metadata and content.

## File Location

All blog posts live in:

```text
src/content/blog/
```

The blog service automatically discovers and processes all `.mdx` files in this directory.

## Naming Convention

Use kebab-case for your blog post filenames. The filename becomes the URL slug.

<Steps>

<Step>

### Choose a descriptive filename

The filename should be lowercase with hyphens separating words:

- `awesome-terminal.mdx` → `/blog/awesome-terminal`
- `print-pdf-in-vercel.mdx` → `/blog/print-pdf-in-vercel`
- `welcome.mdx` → `/blog/welcome`

</Step>

<Step>

### Create the MDX file

Create a new file in `src/content/blog/` with the `.mdx` extension:

```bash
touch src/content/blog/my-new-post.mdx
```

</Step>

<Step>

### Add frontmatter and content

Start with the required frontmatter fields, then add your content below.

</Step>

</Steps>

## Image Handling

### Adding Post Images

Store blog post images in the `public/images/blog/` directory:

```text
public/
└── images/
    └── blog/
        ├── awesome-terminal.webp
        ├── print-pdf-in-vercel.webp
        └── welcome.webp
```

### Referencing Images

Reference images in frontmatter using absolute paths from the public directory:

```yaml
image: /images/blog/my-post-image.webp
```

### Image Best Practices

- **Format**: Use WebP for optimal performance and file size
- **Size**: Recommended 1200x630 pixels for Open Graph images
- **Naming**: Use kebab-case matching your post filename
- **Optimization**: Compress images before adding them to the project

### Using Images in Content

You can also embed images directly in your MDX content:

```mdx
![Alt text](/images/blog/screenshot.webp)
```

Or use the Next.js Image component for advanced optimization:

```mdx
import Image from 'next/image'

<Image
  src="/images/blog/screenshot.webp"
  alt="Screenshot description"
  width={1200}
  height={630}
/>
```

## Automatic Processing

Once you create a blog post, the system automatically:

1. **Discovers** the file in `src/content/blog/`
2. **Parses** frontmatter metadata using gray-matter
3. **Generates** the URL slug from the filename
4. **Sorts** posts by date (newest first, with pinned posts on top)
5. **Creates** static pages at build time
6. **Adds** to RSS feed at `/rss`
7. **Generates** Open Graph images for social sharing

<Note>
The blog service (`src/services/blog.ts`) handles all post processing. No additional configuration is needed when you add new posts.
</Note>
