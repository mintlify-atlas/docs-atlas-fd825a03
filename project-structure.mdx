The Tony Santana Portfolio follows a well-organized Next.js App Router structure with clear separation of concerns. This guide explains the purpose of each directory and key files.

## Root Directory Structure

Here's an overview of the main directories and files:

```
portfolio/
├── src/                    # Source code
├── public/                 # Static assets
├── package.json           # Project dependencies
├── next.config.ts         # Next.js configuration
├── tsconfig.json          # TypeScript configuration
├── biome.json             # Biome linter/formatter config
├── components.json        # shadcn/ui configuration
└── pnpm-lock.yaml        # Lock file for dependencies
```

## Source Directory (`src/`)

The `src/` directory contains all application code, organized by feature and responsibility.

### App Directory (`src/app/`)

The `app/` directory uses Next.js 16 App Router with route groups:

```
src/app/
├── (app)/                 # Main application routes
│   ├── page.tsx          # Portfolio homepage
│   ├── blog/             # Blog listing and posts
│   └── resume/           # Resume pages
├── (llms)/               # LLM-specific routes
├── api/                  # API routes
├── offline/              # PWA offline page
├── print/                # Print-optimized pages
├── vcard/                # vCard download route
├── layout.tsx            # Root layout
├── manifest.ts           # PWA manifest
├── robots.ts             # Robots.txt generation
└── sitemap.ts            # Sitemap generation
```

<Note>
  Route groups like `(app)` and `(llms)` organize routes without affecting the URL structure.
</Note>

## Configuration Files

### Next.js Configuration

**`next.config.ts`**: Configures Next.js behavior, including:
- Image optimization settings
- Webpack configurations
- Redirects and rewrites
- Environment variables

### TypeScript Configuration

**`tsconfig.json`**: TypeScript compiler options with path aliases:

```json tsconfig.json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

This allows imports like `import { Button } from "@/components/ui/button"`.

### Package Management

**`package.json`**: Project metadata and scripts:

```json package.json
{
  "scripts": {
    "dev": "next dev -p 3000",
    "build": "next build",
    "start": "next start",
    "lint": "biome check --write",
    "format": "biome format --write"
  }
}
```

### Component Configuration

**`components.json`**: shadcn/ui configuration:

```json components.json
{
  "style": "default",
  "tailwind": {
    "config": "tailwind.config.ts",
    "css": "src/styles/globals.css"
  },
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils"
  }
}
```

## File Naming Conventions

<Card title="Naming Standards" icon="file">
  - **Components**: PascalCase (e.g., `Button.tsx`, `ThemeToggle.tsx`)
  - **Utilities**: kebab-case (e.g., `use-pwa.ts`, `v-card.ts`)
  - **Routes**: kebab-case directories (e.g., `blog/`, `resume/`)
  - **Config files**: kebab-case with extension (e.g., `site.config.ts`)
</Card>

## Import Paths

The project uses the `@/` alias for cleaner imports:

```typescript
// Instead of:
import { Button } from "../../components/ui/button"

// Use:
import { Button } from "@/components/ui/button"
```

## Key Files to Customize

When adapting this template for your own use:

1. **`src/content/profile.ts`** - Your personal and professional information
2. **`src/config/site.config.ts`** - Site metadata and navigation
3. **`src/content/blog/*.mdx`** - Your blog posts
4. **`public/images/`** - Replace with your own images
5. **`.env.local`** - Your environment variables

<Tip>
  Start by updating the profile data, then customize the styling and components to match your personal brand.
</Tip>
