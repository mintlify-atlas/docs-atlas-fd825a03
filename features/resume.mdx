The Tony Santana Portfolio includes a resume generation system that creates beautiful, professional resumes from MDX files with built-in PDF export functionality.

## Overview

Resumes are stored as MDX files in `src/content/resume/` and rendered at `/resume/[slug]` URLs. The system supports rich formatting, custom styling, and PDF export.

## Creating a Resume

<Steps>
  <Step title="Create MDX file">
    Create a new resume file in `src/content/resume/`:

    ```bash
    touch src/content/resume/your-name.mdx
    ```

    The filename becomes the URL slug: `/resume/your-name`
  </Step>

  <Step title="Add frontmatter">
    Add metadata at the top of your resume file:

    ```mdx
    ---
    title: Tony Santana
    description: Curriculum Vitae of Tony Santana
    createdAt: 2025-09-20
    updatedAt: 2025-09-20
    image: images/resume/tony-santana-cv.webp
    ---
    ```
  </Step>

  <Step title="Write resume content">
    Use MDX to structure your resume with headings, lists, and custom components:

    ```mdx
    <h1 className="text-center pt-2">Your Name</h1>

    <center>
      **Job Title | Specialization**
    </center>

    ## Summary

    Your professional summary here...

    ## Experience

    ### Company Name

    **Position** · Start Date – End Date

    - Achievement or responsibility
    - Another achievement
    ```
  </Step>
</Steps>

## Accessing Resumes

Resumes are accessible via the `/resume/[slug]` route. The page component is defined in `src/app/(app)/(docs)/resume/[slug]/page.tsx:93`:

```typescript
export default async function Page({
  params,
}: {
  params: Promise<{
    slug: string;
  }>
}) {
  const { slug } = await params;
  const cv = getCvBySlug(slug);

  if (!cv) {
    notFound();
  }

  const mdx = cvToMdx(PROFILE);

  return (
    <>
      <Prose className="px-4 xl:px-16">
        <div id="mdx-print-root">
          <MDX code={mdx} />
        </div>
      </Prose>
    </>
  );
}
```

### URL Pattern

```
https://yoursite.com/resume/tony-santana
```

The slug corresponds to the filename without extension.

## PDF Export

The resume page includes a built-in PDF export button powered by the `ExportButton` component.

### Export Button Implementation

From `src/app/(app)/(docs)/resume/[slug]/_components/export-button.tsx:31`:

```typescript
const downloadAsPdf = async () => {
  if (isLoading) return;

  setIsLoading(true);

  try {
    const toastId = toast.loading("Exporting PDF...");

    if (!slug) {
      throw new Error("No document slug provided");
    }

    const element = document.createElement("a");
    element.href = `/tony-santana.pdf`;
    element.download = `${fileName}.pdf`;
    document.body.appendChild(element);
    element.click();
    document.body.removeChild(element);

    toast.success("PDF ready and downloaded", {
      id: toastId,
      description: fileName,
    });
  } catch (error) {
    console.error("PDF export error:", error);
    toast.error("Export failed");
  } finally {
    setIsLoading(false);
  }
};
```

## Resume Metadata

The resume system generates comprehensive metadata for SEO:

```typescript
export async function generateMetadata({
  params,
}: {
  params: Promise<{ slug: string }>;
}): Promise<Metadata> {
  const slug = (await params).slug;
  const cv = getCvBySlug(slug);

  if (!cv) {
    return notFound();
  }

  const { title, description, image, createdAt, updatedAt } = cv.metadata;
  const cvUrl = getCvUrl(cv);
  const ogImage = image || `/og/simple?title=${encodeURIComponent(title)}`;

  return {
    title: "Resume",
    description,
    openGraph: {
      url: cvUrl,
      type: "article",
      images: {
        url: ogImage,
        width: 1200,
        height: 630,
        alt: title,
      },
    },
  };
}
```

## Dynamic Resume Generation

The system can also generate resumes dynamically from profile data using the `cvToMdx()` function:

```typescript
import { cvToMdx } from "@/lib/cv-to-mdx";
import { PROFILE } from "@/content/profile";

const mdx = cvToMdx(PROFILE);
```

This converts profile information into MDX format automatically.

## Best Practices

<Steps>
  <Step title="Keep it concise">
    Aim for 1-2 pages. Focus on relevant experience and achievements.
  </Step>

  <Step title="Use action verbs">
    Start bullet points with strong action verbs: "Led," "Designed," "Implemented."
  </Step>

  <Step title="Quantify achievements">
    Include numbers and percentages: "Increased performance by 70%."
  </Step>

  <Step title="Update regularly">
    Keep your resume current by updating the `updatedAt` field when making changes.
  </Step>

  <Step title="Test PDF export">
    Always test the PDF export to ensure proper formatting and completeness.
  </Step>
</Steps>
