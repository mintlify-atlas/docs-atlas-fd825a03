## Contribution Graph

The contribution graph displays your GitHub activity over the past year.

### Implementation

The contribution data is fetched from `src/content/github-contributions.ts:8`:

```typescript
import type { Activity } from "@/components/shared/contribution-graph";
import { PROFILE } from "@/content/profile";

type GitHubContributionsResponse = {
  contributions: Activity[];
};

export async function getGitHubContributions() {
  const res = await fetch(
    `https://github-contributions-api.jogruber.de/v4/${PROFILE.githubUserName}?y=last`,
    {
      next: { revalidate: 86400 }, // Cache for 1 day (86400 seconds)
    }
  );
  const data = (await res.json()) as GitHubContributionsResponse;

  return data.contributions;
}
```

### API Endpoint

The integration uses the public GitHub Contributions API:

```
https://github-contributions-api.jogruber.de/v4/{username}?y=last
```

**Query Parameters:**
- `{username}` - Your GitHub username
- `?y=last` - Fetch contributions from the last year

<Note>
  This is a third-party API that provides contribution data without requiring authentication.
</Note>

### Caching Strategy

Contribution data is cached for performance:

```typescript
{
  next: { revalidate: 86400 } // Cache for 24 hours
}
```

This reduces API calls and improves page load times while keeping data reasonably fresh.

## Activity Data Structure

The API returns an array of activity objects:

```typescript
type Activity = {
  date: string;           // ISO date string (YYYY-MM-DD)
  count: number;          // Number of contributions
  level: 0 | 1 | 2 | 3 | 4; // Contribution intensity level
};
```

### Example Response

```json
{
  "contributions": [
    {
      "date": "2025-03-04",
      "count": 12,
      "level": 3
    },
    {
      "date": "2025-03-03",
      "count": 5,
      "level": 2
    }
  ]
}
```

### Contribution Levels

- `level: 0` - No contributions
- `level: 1` - Low activity (1-3 contributions)
- `level: 2` - Moderate activity (4-6 contributions)
- `level: 3` - High activity (7-9 contributions)
- `level: 4` - Very high activity (10+ contributions)

## Using the Contribution Graph Component

To display the contribution graph in your portfolio:

```tsx
import { ContributionGraph } from "@/components/shared/contribution-graph";
import { getGitHubContributions } from "@/content/github-contributions";

  const contributions = await getGitHubContributions();

  return (
    <div>
      <h2>GitHub Contributions</h2>
      <ContributionGraph data={contributions} />
    </div>
  );
}
```

## GitHub API Token (Optional)

For advanced features or higher rate limits, you can configure a GitHub Personal Access Token.

### Environment Variable

Set the `GITHUB_API_TOKEN` in your environment:

```bash
GITHUB_API_TOKEN=ghp_your_token_here
```

### Getting a Token

<Steps>
  <Step title="Visit GitHub Settings">
    Go to [GitHub Settings > Tokens](https://github.com/settings/tokens)
  </Step>

  <Step title="Generate new token">
    Click "Generate new token (classic)"
  </Step>

  <Step title="Set permissions">
    For basic contribution data, no special scopes are needed.

    For repository information, select:
    - `public_repo` - Access public repositories
  </Step>

  <Step title="Copy token">
    Copy the generated token and add it to your `.env.local` file
  </Step>
</Steps>

<Note>
  The contribution graph API doesn't require authentication. The token is only needed for features like repository stars, detailed commit information, or higher rate limits.
</Note>

## Customization

### Change Time Range

Modify the API query to fetch different time ranges:

```typescript
// Last year (default)
`https://github-contributions-api.jogruber.de/v4/${username}?y=last`

// Specific year
`https://github-contributions-api.jogruber.de/v4/${username}?y=2024`

// All years
`https://github-contributions-api.jogruber.de/v4/${username}`
```

### Adjust Cache Duration

Change the revalidation period:

```typescript
{
  next: { revalidate: 3600 }  // 1 hour
}

{
  next: { revalidate: 43200 }  // 12 hours
}

{
  next: { revalidate: 86400 }  // 24 hours (default)
}
```

## Troubleshooting

<Steps>
  <Step title="No data showing">
    Verify your `githubUserName` is correctly set in `src/content/profile.ts`
  </Step>

  <Step title="API rate limiting">
    The public API has rate limits. If exceeded, add a `GITHUB_API_TOKEN` environment variable.
  </Step>

  <Step title="Stale data">
    Clear Next.js cache:

    ```bash
    rm -rf .next/cache
    pnpm dev
    ```
  </Step>

  <Step title="Build errors">
    Ensure the API is accessible during build time. Consider using static fallback data.
  </Step>
</Steps>
