This guide walks you through setting up your local development environment for the Tony Santana Portfolio project.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** 20.x or later
- **pnpm** (recommended package manager)
- **Git** for version control

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the project to your local machine:

    ```bash
    git clone https://github.com/your-username/portfolio.git
    cd portfolio
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages using pnpm:

    ```bash
    pnpm install
    ```

    <Note>
      This project uses **pnpm** as the package manager. While npm and yarn may work, pnpm is recommended for consistency with the project configuration.
    </Note>
  </Step>

  <Step title="Set up environment variables">
    Copy the example environment file and configure your variables:

    ```bash
    cp .env.example .env.local
    ```

    Update the `.env.local` file with your configuration. See the [Environment Variables](/configuration/environment-variables) guide for details.
  </Step>

  <Step title="Start the development server">
    Run the development server:

    ```bash
    pnpm dev
    ```

    Your application will be available at `http://localhost:3000`.
  </Step>
</Steps>

## Available Scripts

The project includes several npm scripts for development and production workflows:

<CodeGroup>
```bash dev
# Start development server on port 3000
pnpm dev
```

```bash build
# Create production build
pnpm build
```

```bash start
# Start production server
pnpm start
```

```bash lint
# Run Biome linter and auto-fix issues
pnpm lint
```

```bash format
# Format code with Biome
pnpm format
```

```bash build:debug
# Build with debug prerender information
pnpm build:debug
```
</CodeGroup>

### Script Details

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `next dev -p 3000` | Starts the Next.js development server on port 3000 with hot reload |
| `build` | `next build` | Creates an optimized production build |
| `build:debug` | `next build --debug-prerender` | Builds with detailed prerender debugging information |
| `start` | `next start` | Starts the production server (run after `build`) |
| `lint` | `biome check --write` | Runs linting and auto-fixes issues |
| `format` | `biome format --write` | Formats code according to Biome configuration |

## Development Workflow

### Hot Reload

The development server includes hot module replacement (HMR). Changes to your code will automatically reflect in the browser without requiring a full page reload:

- **React components** - Instant updates while preserving state
- **CSS/Tailwind** - Live style updates
- **API routes** - Automatic server restart
- **Configuration files** - Requires manual server restart

<Tip>
  For configuration changes in `next.config.mjs`, `tailwind.config.ts`, or `biome.json`, you'll need to restart the development server manually.
</Tip>

### Local Development Tips

1. **Fast Refresh**: Next.js preserves component state during edits. If you encounter issues, try refreshing the browser.

2. **Port Configuration**: The dev server runs on port 3000 by default. To use a different port:
   ```bash
   pnpm dev -- -p 4000
   ```

3. **TypeScript Checking**: The dev server doesn't block on TypeScript errors. Run type checking separately:
   ```bash
   npx tsc --noEmit
   ```

4. **Clear Cache**: If you encounter build issues, try clearing the Next.js cache:
   ```bash
   rm -rf .next
   pnpm dev
   ```

## Debugging

### Browser DevTools

Use browser developer tools for debugging:

- **React DevTools** - Install the browser extension for React component inspection
- **Console Logging** - Use `console.log()` for quick debugging
- **Source Maps** - Development builds include source maps for easy debugging

### VS Code Debugging

Create a `.vscode/launch.json` file for integrated debugging:

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Next.js: debug server-side",
      "type": "node-terminal",
      "request": "launch",
      "command": "pnpm dev"
    },
    {
      "name": "Next.js: debug client-side",
      "type": "chrome",
      "request": "launch",
      "url": "http://localhost:3000"
    }
  ]
}
```

### Debug Prerendering

To debug prerendering issues, use the debug build:

```bash
pnpm build:debug
```

This provides detailed information about static generation and server-side rendering.
