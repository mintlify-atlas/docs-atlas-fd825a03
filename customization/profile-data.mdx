The `PROFILE` object in `src/content/profile.ts` is the central configuration for your entire portfolio. It contains all your personal information, work experience, projects, tech stack, and more.

## Profile Structure

The profile is structured with basic information and multiple sections:

```typescript
export const PROFILE: IProfile = {
  // Basic Information
  firstName: "Tony",
  lastName: "Santana",
  displayName: "Tony Santana",
  username: "tonysantana1492",
  bio: "Creating with code. Small details matter.",
  
  // Contact Information (base64 encoded)
  email: "base64_encoded_email",
  phoneNumber: "base64_encoded_phone",
  
  // Social Profiles
  githubUserName: "tonysantana1492",
  twitterUsername: "tonysantana1492",
  linkedinUserName: "tonysantana1492",
  
  // Sections
  sections: {
    about: { /* ... */ },
    experiences: { /* ... */ },
    projects: { /* ... */ },
    techStack: { /* ... */ },
    // ... more sections
  }
}
```

</ParamField>

<ParamField path="avatar" type="string" required>
  URL or path to your profile photo
</ParamField>

<ParamField path="ogImage" type="string" required>

</ParamField>

## Contact Information

<Warning>
  For privacy, email and phone number should be base64 encoded. Use a [base64 encoder](https://www.base64encode.org/) to encode your contact information.
</Warning>

<ParamField path="email" type="string" required>
  Your email address (base64 encoded)
  
  ```typescript
  // Example: "tony@example.com" becomes:
  email: "dG9ueUBleGFtcGxlLmNvbQ=="
  ```
</ParamField>

<ParamField path="phoneNumber" type="string">
  Your phone number in E.164 format, base64 encoded
  
  ```typescript
  // Example: "+1234567890" becomes:
  phoneNumber: "KzEyMzQ1Njc4OTA="
  ```
</ParamField>

<ParamField path="address" type="string">
  Your location or address
</ParamField>

<ParamField path="website" type="string" required>
  Your portfolio website URL
</ParamField>

## Portfolio Sections

Each section follows a common structure:

```typescript
interface Section<T> {
  id: string;           // Unique identifier for the section
  name: string;         // Display name
  columns: number;      // Number of columns for layout (1-3)
  content?: string;     // Optional description
  visible: boolean;     // Show/hide the section
  separateLinks: boolean; // Whether to separate links
  items: T[];          // Array of section items
  icon: string;        // Icon name (Lucide icon)
}
```

### Available Sections

- **about** - About me information
- **experiences** - Work experience
- **educations** - Education history
- **projects** - Portfolio projects
- **techStack** - Technologies and tools
- **certifications** - Professional certifications
- **awards** - Awards and achievements
- **socialLinks** - Social media profiles

<Tip>
  See [Portfolio Sections](/customization/portfolio-sections) for detailed configuration of each section type.
</Tip>

## Example Configuration

Here's a complete example for the experiences section:

```typescript
experiences: {
  id: "experiences",
  name: "Experience",
  columns: 1,
  visible: true,
  separateLinks: false,
  icon: "briefcase",
  items: [
    {
      id: "officepuzzle",
      companyName: "OfficePuzzle",
      companyLogo: "/images/companies/officepuzzle.svg",
      isCurrentEmployer: true,
      website: "https://officepuzzle.com",
      positions: [
        {
          id: "senior-engineer",
          title: "Senior Software Engineer",
          employmentType: "Full-time",
          location: "Remote",
          employmentPeriod: {
            start: "2023-01",
            end: undefined // Current position
          },
          description: "Leading development of healthcare platform...",
          skills: ["Next.js", "TypeScript", "PostgreSQL"],
          isExpanded: false
        }
      ]
    }
  ]
}
```
