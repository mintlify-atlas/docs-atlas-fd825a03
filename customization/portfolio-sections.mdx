Portfolio sections are the building blocks of your profile. Each section displays different types of content like work experience, projects, tech stack, and more.

## Section Structure

All sections follow a common structure:

```typescript
interface Section<T> {
  id: string;              // Unique identifier (used in URLs)
  name: string;            // Display name
  columns: number;         // Layout columns (1-3)
  content?: string;        // Optional description
  visible: boolean;        // Show/hide section
  separateLinks: boolean;  // Separate item links
  items: T[];             // Section content
  icon: string;           // Lucide icon name
}
```

## Available Sections

### About Section

Display personal information and interests:

```typescript
about: {
  id: "about",
  name: "About",
  columns: 2,
  visible: true,
  separateLinks: false,
  icon: "user",
  items: [
    {
      id: "intro",
      title: "Introduction",
      icon: "circle-user",
      content: "I'm a full-stack developer with 7+ years of experience..."
    },
    {
      id: "interests",
      title: "Interests",
      icon: "heart",
      content: "Open source, web performance, developer tooling"
    }
  ]
}
```

### Experience Section

Showcase your work history:

```typescript
experiences: {
  id: "experiences",
  name: "Experience",
  columns: 1,
  visible: true,
  separateLinks: false,
  icon: "briefcase",
  items: [
    {
      id: "company-1",
      companyName: "OfficePuzzle",
      companyLogo: "/images/companies/officepuzzle.svg",
      isCurrentEmployer: true,
      website: "https://officepuzzle.com",
      positions: [
        {
          id: "position-1",
          title: "Senior Software Engineer",
          employmentType: "Full-time",
          location: "Remote",
          employmentPeriod: {
            start: "2023-01",
            end: undefined // Current
          },
          description: "Leading development of healthcare platform with Next.js and PostgreSQL...",
          skills: ["Next.js", "TypeScript", "PostgreSQL", "AWS"],
          isExpanded: false
        }
      ]
    }
  ]
}
```

<Note>
  You can have multiple positions per company. Set `isCurrentEmployer: true` for your current role.
</Note>

### Tech Stack Section

Display your technology expertise:

```typescript
techStack: {
  id: "tech-stack",
  name: "Tech Stack",
  columns: 3,
  visible: true,
  separateLinks: false,
  icon: "layers",
  items: [
    {
      key: "react",
      title: "React",
      href: "https://react.dev",
      categories: ["Frontend", "Library"],
      theme: true // Use dark/light variants
    },
    {
      key: "nextjs",
      title: "Next.js",
      href: "https://nextjs.org",
      categories: ["Frontend", "Framework"],
      theme: true
    }
  ]
}
```

<Tip>
  Set `theme: true` to use different icons for light and dark modes. Place icons in `public/tech-stack-icons/` as:
  - `{key}.svg` (default)
  - `{key}-dark.svg` (dark mode)
  - `{key}-light.svg` (light mode)
</Tip>

## Section Configuration Options

### Visibility

Control which sections appear on your portfolio:

```typescript
visible: true  // Show section
visible: false // Hide section
```

### Column Layout

Adjust the number of columns (1-3):

```typescript
columns: 1 // Single column (good for detailed content)
columns: 2 // Two columns (balanced layout)
columns: 3 // Three columns (compact, grid-like)
```

<Note>
  Mobile devices always display single column layout regardless of this setting.
</Note>

### Section Icons

Use any [Lucide icon](https://lucide.dev/icons) name:

```typescript
icon: "briefcase"      // Work experience
icon: "folder"         // Projects
icon: "graduation-cap" // Education
icon: "award"          // Certifications
icon: "trophy"         // Awards
icon: "layers"         // Tech stack
```

## Reordering Sections

The order of sections in your profile object determines their display order:

```typescript
sections: {
  about: { /* ... */ },        // Shows first
  experiences: { /* ... */ },  // Shows second
  projects: { /* ... */ },     // Shows third
  techStack: { /* ... */ },    // Shows fourth
  // ... more sections
}
```

Simply reorder the object keys to change section order.

## Best Practices

<Accordion title="Keep content concise">
  Use brief descriptions. Visitors can expand items for more details if interested.
</Accordion>

<Accordion title="Use high-quality images">
  Upload company logos and project screenshots at 2x resolution for retina displays.
</Accordion>

<Accordion title="Update regularly">
  Keep your experience, projects, and skills current to reflect your latest work.
</Accordion>

<Accordion title="Prioritize important sections">
  Place your strongest sections (usually experience and projects) near the top.
</Accordion>

<Accordion title="Use consistent date formats">
  Use YYYY-MM format for dates (e.g., "2024-03") for proper sorting and formatting.
</Accordion>
